A Farbrausch software archive.

Date: Jan-10-2021.

Farbrausch is a german demoscene group, they are popular for creating lightweight but amazing demoscenes.

They created a lot of stuff including: Kkrieger (A 3D game achieved in less than 100 kilobytes), Werkkzeug (engines for creating lightweight demos and textures) and a lot more.

Ultimately, a lot of its members just stopped putting out new content and they decided in 2011 to release all of the source code for all of their projects.

Some of their members went on to still be active in the demoscene community or just went on with their lives.

I was so interested about Farbrausch in 2019, I decided to archive all software I could find that was associated with them

However, due to an error, I accidentally formatted my USB drive and lost a lot of important things.

so in 2021, I decided to archive all of their software and make it available in a big pile of archives.

Downloads for the demos.7z file are temporarily down.

Github's LFS limit prevents me from uploading demos.7z, you can download ultra.7z though...

There are 2 important files:

  • files.txt - Explaining what each app does and where I got it from (The Origin)
  • ultra.7z - Software collection, Size: 32 MB.
  • demos.7z - Demos collection, Size 295MB.

The archives are compressed using 7zip's Ultra settings.

I remember watching a video about a 96KB video game (Kkrieger) made by Farbrausch, I was very interested in how it worked, it was really amazing to see how much a 96KB game could do! I thought that since they are an old group and their website isn't very popular, it might close sometime soon. So I decided to archive everything I could find, with all of that being said, there were some missing pieces.

Compression techniques

Here is a list of techniques I used to get the files as small as possible.

  • Extracting every single compressed thing and them compressing it again

    • If you are compressing something with a lot of compressed archives then chances are that your archive won't get smaller, It's because the archives were already compressed and so the algorithm that you're using might not be able to do much, so please extract everything that you can and then compress it. it will save some extra space!

  • Using maximum compression settings

    • I used 7zip because it supports a lot of different algorithms and settings, the exact switches I used were
    -t7z -mx=9 -mfb=273 -ms -md=31 -myx=9 -mtm=- -mmt -mmtf -md=1536m -mmf=bt3 -mmc=10000 -mpb=0 -mlc=0, so the Linux command was
    7z a -t7z -mx=9 -mfb=273 -ms -md=31 -myx=9 -mtm=- -mmt -mmtf -md=1536m -mmf=bt3 -mmc=10000 -mpb=0 -mlc=0 ARCHIVENAME.7z folder folder/*
